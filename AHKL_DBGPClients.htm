Debugger-Clients

DBGp ist ein häufig verwendetes Debugger-Protokoll für Programmiersprachen und Debugger-UI-Kommunikation. Mit diesem Protokoll kann man auf erweiterte Funktionen zur Fehleranalyse zugreifen. Siehe Interaktives Debugging für mehr Details. Auf dieser Seite sind einige Benutzeroberflächen oder "Clients" aufgelistet, die kompatibel mit AutoHotkey sind.

SciTE4AutoHotkey

SciTE4AutoHotkey ist ein freier, SciTE-basierter Editor für AutoHotkey-Skripte. Dieser Editor unterstützt neben DBGp auch Syntaxhervorhebung, Calltips/Parameterinformation, automatische Vervollständigung und weitere nützliche Bearbeitungsfunktionen und Skript-Werkzeuge.

Folgende Funktionen sind zur Fehleranalyse enthalten:

  • Haltepunkte.
  • Ausführen, Betreten, Überspringen, Verlassen.
  • Aufrufliste anzeigen.
  • Namen und Inhalte von lokalen/globalen Variablen auflisten.
  • Variableninhalte anzeigen, wenn sich der Mauszeiger darüber befindet.
  • Variableninhalte überwachen und bearbeiten.
  • Strukturen von Objekten anzeigen.

XDebugClient

XDebugClient ist ein einfacher Open-Source-Frontend-DBGp-Client auf Basis von .NET Framework 2.0. XDebugClient wurde ursprünglich für PHP mit Xdebug entwickelt, jedoch ist ein benutzerdefinierter Build weiter unten erhältlich, der mit AutoHotkey kompatibel ist.

Änderungen:

  • Die Debugger-Engine kann neben "php" auch andere Sprachen benachrichtigen.
  • Syntax-Hervorhebung für AutoHotkey hinzugefügt.
  • Wartet automatisch auf eine Verbindungsanfrage von der Debugger-Engine, anstatt darauf zu warten, bis der Benutzer Start Listening anklickt.
  • Eigenschaftswerte werden beim ersten Null-Zeichen gekürzt, da AutoHotkey derzeit den gesamten Variableninhalt zurückgibt und XDebugClient keine geeignete Schnittstelle für die Anzeige von binären Inhalten hat.

Download: Binärdateien; Quellcode (siehe auch SharpDevelop, Dockpanel Suite und Advanced Treeview)

Verwendung:

  • Starte XDebugClient.
  • Starte AutoHotkey /Debug. XDebugClient sollte automatisch die Skript-Datei öffnen.
  • Klicke den linken Rand an, um zumindest einen Haltepunkt zu setzen.
  • Wähle Run im Debug-Menü aus, oder drücke F5.
  • Sobald die Ausführung den Haltepunkt erreicht, kann man das Debug-Menü oder die Tastenkürzel verwenden, um das Skript schrittweise durchzugehen oder fortzusetzen.

Features:

  • Schreibgeschützte Ansicht des Quellcodes mit Syntaxhervorhebung.
  • Haltepunkte.
  • Ausführen, Betreten, Überspringen, Verlassen.
  • Aufrufliste anzeigen.
  • Variablen überprüfen - Variablenname markieren, Rechtsklick, Inspect.

Probleme:

  • Die Benutzeroberfläche reagiert nicht auf die Benutzereingaben, während das Skript läuft.
  • Keine Mechanismen verfügbar, Variablen aufzulisten oder deren Werte zu setzen.

DBGp-Plugin für Notepad++

Ein DBGp-Client ist als Plugin für Notepad++ verfügbar. Obwohl es für PHP entwickelt wurde, funktioniert es auch mit AutoHotkey.

Download: Siehe Verzeichnis für Notepad++-Plugins.

Verwendung:

  • Starte Notepad++.
  • Konfiguriere das DBGp-Plugin über Erweiterungen, DBGp, Config...

    Hinweis: Dateizuordnung muss konfiguriert werden. Die meisten Benutzer werden keine ferngesteuerte Fehleranalyse durchführen, deshalb kann man einfach ein Häkchen bei Bypass all mapping (local windows setup) setzen.

  • Zeige das Debugger-Fenster per Werkzeugleiste oder Erweiterungen, DBGp, Debugger an.
  • Öffne das Skript, das analysiert werden soll.
  • Setze mindestens einen Haltepunkt.
  • Starte AutoHotkey /Debug.
  • Verwende die Debugger-Werkzeugleiste oder Tastenkürzel, um den Debugger zu steuern.

Features:

  • Syntaxhervorhebung, wenn es vom Benutzer konfiguriert wird.
  • Haltepunkte.
  • Ausführen, Betreten, Überspringen, Verlassen, Bis zum Cursor ausführen, Stop.
  • Lokale/globale Variablen anzeigen.
  • Benutzerdefinierte Variablen überwachen.
  • Aufrufliste anzeigen.
  • Variableninhalte anzeigen, wenn sich der Mauszeiger darüber befindet.
  • Frei konfigurierbare Tastenkombinationen - Einstellungen, Tastatur ..., Plugin commands.

Probleme:

  • Der Inhalt einer Variable kann nicht mit der Maus angezeigt werden, wenn der Variablenname nur aus einem Buchstaben besteht - Bei dem Variablennamen "a" beispielsweise versucht das Plugin, " a" oder "a " abzurufen.
  • Befindet sich der Mauszeiger über ein Text, versucht das Plugin, eine Variable abzurufen, selbst wenn der Text ungültige Zeichen enthält.
  • Notepad++ wird instabil, wenn property_get fehlschlägt, was besonders problematisch im Bezug auf den oberen Punkt ist. Als Übergangslösung sendet AutoHotkey eine leere Eigenschaft anstelle eines Fehlercodes, wenn eine nicht vorhandene oder ungültige Variable angefordert wird.

Skript-basierte Clients

Auf GitHub steht eine skript-basierte DBGp-Library und Beispiel-Clients zur Verfügung.

  • dbgp_console.ahk: Einfacher Kommandozeilen-Client.
  • dbgp_test.ahk: Zeigt asynchrone Fehleranalyse.
  • dbgp_listvars.ahk: Ein Beispiel-Client, der die Variablen von allen laufenden Skripten auflistet.

GitHub: Lexikos / dbgp

Kommandozeilen-Client

Ein Kommandozeilen-Client ist bei xdebug.org erhältlich. Dieser Client ist für die meisten Benutzer allerdings nicht geeignet, weil es Kenntnisse über DBGp (das Protokoll) erfordert.

Sonstige

Es stehen eine Reihe von anderen DBGp-Clients zur Verfügung, die jedoch noch nicht mit AutoHotkey getestet wurden. Siehe Xdebug: Dokumentation.
