#ClipboardTimeout

Ändert, wie lange das Skript versuchen soll, auf die Zwischenablage zuzugreifen, falls der erste Versuch fehlschlägt.

#ClipboardTimeout Millisekunden

Parameter

Millisekunden

Die Länge des Intervalls in Millisekunden. Mit -1 wird der Versuch unendlich oft durchgeführt. Mit 0 wird der Versuch nur einmal durchgeführt. Skripte, die diese Direktive nicht enthalten, verwenden ein Zeitlimit von 1000 ms.

Bemerkungen

Einige Anwendungen halten die Zwischenablage für einen langen Zeitraum offen, um zum Beispiel eine große Datenmenge schreiben oder lesen zu können. In solchen Fällen bewirkt das Erhöhen dieser Einstellung, dass das Skript länger wartet, bevor es aufgibt und eine Fehlermeldung anzeigt.

Diese Einstellungen gelten für alle Clipboard-Operationen, die einfachsten davon sind folgende Beispiele: Var := Clipboard und Clipboard := "Neuer Text".

Während das Skript auf die Verfügbarkeit der Zwischenablage wartet, ist es nicht möglich, dass neue Threads gestartet oder Timer laufen können. Wenn der Benutzer allerdings einen Hotkey drückt, einen benutzerdefinierten Menüpunkt auswählt oder eine GUI-Aktion durchführt, wie z. B. das Drücken eines Buttons, wird dieses Ereignis für später zwischengespeichert. Das heißt, dass die Subroutine des Buttons erst ausgeführt wird, wenn die Zwischenablage wieder verfügbar ist.

Diese Direktive bewirkt nicht, dass die Daten der Zwischenablage noch einmal gelesen werden, wenn der erste Versuch fehlschlägt. Vor v1.1.16 bewirkte sie, dass das Skript auf das Ablaufen des Timeouts wartete, um so zu verhindern, dass weiteren Daten abgerufen werden konnten.
