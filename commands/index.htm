Alphabetischer Funktionsindex

Klicke auf einen Funktionsnamen, um mehr zu erfahren. Häufig verwendete Einträge sind groß geschrieben.

Eintrag Beschreibung
{ ... } (Block) Zwei geschweifte Klammern kennzeichnen einen Block. Blöcke werden typischerweise bei Funktionen, Else, Loop, While-Schleifen und If-Anweisungen verwendet.
{ ... } / Object Erstellt ein skriptfähiges assoziatives Array.
[ ... ] / Array Erstellt ein skriptfähiges assoziatives Array mit Integer-Keys.
Abs Gibt den absoluten Wert von Zahl zurück.
ASin Gibt den Arkussinus (die Zahl, deren Sinus Zahl ist) als Bogenmaß zurück.
ACos Gibt den Arkuskosinus (die Zahl, deren Kosinus Zahl ist) als Bogenmaß zurück.
ATan Gibt den Arkustangens (die Zahl, deren Tangens Zahl ist) als Bogenmaß zurück.
BlockInput Deaktiviert oder aktiviert die Fähigkeit des Benutzers, mit dem Computer mittels Tastatur und Maus zu interagieren.
Break Beendet bzw. verlässt eine Schleife. Nur in einer Schleife gültig.
Catch Bestimmt den Code, der ausgeführt werden soll, wenn eine Ausnahme während einer Try-Anweisung ausgelöst wird.
Ceil Gibt Zahl zurück, welche auf den nächsten Integer aufgerundet ist (ohne .00-Suffix).
Chr Gibt eine Zeichenkette zurück (üblicherweise ein einzelnes Zeichen), deren Zeichencode mit der angegebenen Nummer übereinstimmt.
Click Klickt mit einer Maustaste auf bestimmte Koordinaten. Kann auch eine Maustaste gedrückt halten, das Mausrad drehen oder die Maus bewegen.

WinGetList Ermittelt die eindeutigen ID-Nummern von allen Fenstern, mit dem angegebenen Fenster übereinstimmen.
WinGetMinMax Ermittelt den minimierten/maximierten Zustand des angegebenen Fensters.
WinGetPID / WinGetProcessName / WinGetProcessPath Ermittelt die ID, den Namen oder den vollständigen Pfad des Prozesses, zu dem das angegebene Fenster gehört.
WinGetPos Ermittelt die Position und Größe eines bestimmten Fensters.
WinGetStyle Ermittelt einen Integer, der den Style des angegebenen Fensters repräsentiert.
WinGetText Ermittelt den Text des angegebenen Fensters.
WinGetTitle Ermittelt den Titel eines bestimmten Fensters.
WinGetTransColor Ermittelt die Farbe, die im angegebenen Fenster als transparent markiert wurde, sofern es eine gibt.
WinGetTransparent Ermittelt die Transparenzstärke des angegebenen Fensters, sofern vorhanden.
WinHide Versteckt das angegebene Fenster.
WinKill Zwingt das angegebene Fenster sich zu schließen.
WinMaximize Vergrößert das angegebene Fenster auf dessen maximale Größe.
WinMinimize Blendet das angegebene Fenster aus und verschiebt es in die Taskleiste.
WinMinimizeAll Minimiert alle Fenster.
WinMinimizeAllUndo Kehrt die Wirkung von WinMinimizeAll um.
WinMove Verändert die Position und/oder Größe des angegebenen Fensters.
WinMoveBottom Sendet das Fenster an das untere Ende des Stapels, so dass es unter allen anderen Fenstern liegt.
WinMoveTop Setzt ein Fenster an die Spitze des Stapels, ohne es explizit zu aktivieren.
WinRedraw Versucht, die Darstellung/Inhalte eines Fensters zu aktualisieren.
WinRestore Hebt die Minimierung oder Maximierung des angegebenen Fensters auf, falls es minimiert oder maximiert ist.
WinSetAlwaysOnTop Ändert ein Fenster, so dass es immer über allen anderen Fenstern (außer andere Immer-Im-Vordergrund-Fenster) liegt.
WinSetEnabled Aktiviert oder deaktiviert ein Fenster.
WinSetExStyle Ändert den erweiterten Style eines Fensters.
WinSetRegion Ändert die Form eines Fensters zu einem Rechteck, Polygon oder einer Ellipse.
WinSetStyle Ändert den Style eines Fensters.
WinSetTitle Ändert den Titel des angegebenen Fensters.
WinSetTransColor Macht alle Pixel der ausgewählten Farbe innerhalb des Zielfensters unsichtbar.
WinSetTransparent Macht ein Fenster halbtransparent.
WinShow Macht das angegebene Fenster wieder sichtbar.
WinWait Wartet, bis das angegebene Fenster existiert.
WinWaitActive Wartet, bis das angegebene Fenster aktiv ist.
WinWaitClose Wartet, bis das angegebene Fenster nicht mehr existiert.
WinWaitNotActive Wartet, bis das angegebene Fenster nicht mehr aktiv ist.
#ClipboardTimeout Ändert, wie lange das Skript versuchen soll, auf die Zwischenablage zuzugreifen, falls der erste Versuch fehlschlägt.
#ErrorStdOut Sendet alle Syntaxfehler, die den Start eines Skripts verhindern, zur Standardausgabe, anstatt ein Dialogfenster anzuzeigen.
#HotkeyInterval In Verbindung mit #MaxHotkeysPerInterval bestimmt diese Direktive, wie viele Hotkeys innerhalb eines Zeitrahmens aktiviert werden können, bevor eine Warnmeldung erscheint.
#HotkeyModifierTimeout Beeinflusst das Verhalten der Hotkey-Modifikatoren: STRG, ALT, WIN und UMSCHALT.
#Hotstring Ändert Hotstring-Optionen oder Endungszeichen.
#If Vergleichbar mit #IfWinActive, aber für beliebige Ausdrücke.
#IfTimeout Bestimmt die maximale Zeit, die zur Auswertung eines einzelnen #If-Ausdrucks aufgebracht werden soll.
#IfWinActive / #IfWinExist Erstellt kontextabhängige Hotkeys und Hotstrings. Solche Hotkeys können je nach Typ des aktiven oder vorhandenen Fensters unterschiedliche (oder gar keine) Aktionen ausführen.
#Include Veranlasst das Skript sich so zu verhalten, als würde sich der Inhalt der angegebenen Datei direkt an dieser Position befinden.
#InputLevel Bestimmt, welche künstlichen Tastatur- oder Mausereignisse von Hotkeys und Hotstrings ignoriert werden sollen.
#InstallKeybdHook Erzwingt die bedingungslose Installation des Tastatur-Hooks.
#InstallMouseHook Erzwingt die bedingungslose Installation des Maus-Hooks.
#KeyHistory Bestimmt die maximale Anzahl an Tastatur- und Mausereignissen, die im KeyHistory-Fenster angezeigt werden sollen. Mit 0 kann der Tastenverlauf deaktiviert werden.
#MaxHotkeysPerInterval In Verbindung mit #HotkeyInterval bestimmt diese Direktive, wie viele Hotkeys innerhalb eines Zeitrahmens aktiviert werden können, bevor eine Warnmeldung erscheint.
#MaxThreads Bestimmt die maximale Anzahl an gleichzeitig vorhandenen Threads.
#MaxThreadsBuffer Bringt einige oder alle Hotkeys dazu, Tastatureingaben zwischenzuspeichern anstatt zu ignorieren, wenn ihr Grenzwert von #MaxThreadsPerHotkey erreicht wurde.
#MaxThreadsPerHotkey Bestimmt die maximale Anzahl an gleichzeitig vorhandenen Threads pro Hotkey oder Hotstring.
#MenuMaskKey Ändert die Taste, mit der WIN- oder ALT-KeyUp-Ereignisse maskiert werden sollen.
#NoTrayIcon Deaktiviert die Darstellung des Infobereichssymbols.
#Persistent Hält ein Skript dauerhaft aufrecht (bis der Benutzer das Skript schließt oder ein ExitApp vorkommt).
#SingleInstance Bestimmt, ob ein bereits laufendes Skript nochmals ausgeführt werden darf.
#UseHook Erzwingt die Verwendung des Hooks, um alle oder einige Tastatur-Hotkeys zu implementieren.
#Warn Aktiviert oder deaktiviert Warnungen bei bestimmten Zuständen, die ein Fehler sein könnten, wie Schreibfehler oder fehlende "globale" Deklarationen.
#WinActivateForce Erzwingt die gewaltsame Methode zum Aktivieren eines Fensters.
