Suspend

Deaktiviert oder aktiviert alle oder bestimmte Hotkeys und Hotstrings.

Suspend Modus

Parameter

Modus

On oder 1 (true): Sperrt alle Hotkeys und Hotstrings (siehe Bemerkungen zu den Ausnahmen).

Off oder 0 (false): Aktiviert die zuvor deaktivierten Hotkeys und Hotstrings.

Toggle oder -1 (Standardwert): Setzt den Zustand jeweils auf die entgegengesetzte Seite (On oder Off).

Bemerkungen

Standardmäßig kann das Skript auch via Tray-Symbol oder Hauptfenster gesperrt werden.

Um einen Hotkey/Hotstring von einer Sperrung zu befreien, setzt man die #SuspendExempt-Direktive davor. Ein befreiter Hotkey/Hotstring bleibt auch bei eingeschalteter Sperrung aktiviert. Auf diese Weise kann die Sperrung mit einem Hotkey aufgehoben werden, was ansonsten nicht möglich wäre, da der Hotkey dann gesperrt sein würde.

Die Tastatur- und/oder Maus-Hooks werden bei dieser Funktion je nach Bedarf automatisch installiert oder entfernt.

Mit #If können je nach Bedingung bestimmte Hotkeys oder Hotstrings automatisch deaktiviert werden.

Threads, die bereits laufen, können von Suspend nicht gestoppt werden; verwenden Sie Pause, um das zu tun.

Sobald Hotkeys via Suspend gesperrt werden, wird ein S im Tray-Symbol angezeigt. Um das zu verhindern, kann das Symbol "eingefroren" werden, indem eine 1 im letzten Parameter der TraySetIcon-Funktion angegeben wird. Zum Beispiel:

TraySetIcon "C:\Mein Symbol.ico", , 1

Die interne Variable A_IsSuspended enthält eine 1, wenn das Skript mit Suspend gesperrt wurde, ansonsten 0.

Siehe auch

#SuspendExempt, Hotkeys, Hotstrings, #If, Pause, ExitApp

Beispiel

^!s::Suspend  ; Weist einem Hotkey die Umschaltfunktion von Suspend zu.
; Sendet eine Suspend-Funktion zu einem anderen Skript.
DetectHiddenWindows True
WM_COMMAND := 0x111
ID_FILE_SUSPEND := 65404
PostMessage WM_COMMAND, ID_FILE_SUSPEND,,, "C:\DeinSkript.ahk ahk_class AutoHotkey"
