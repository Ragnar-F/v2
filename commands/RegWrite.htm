RegWrite

Schreibt einen Wert in die Registry.

RegWrite Wert, Werttyp, KeyName , Wertname
RegWrite Wert , Werttyp, , Wertname

Parameter

Wert

Der Wert, der geschrieben werden soll. Lange Textwerte können in mehreren kurzen Zeilen aufgeteilt werden (Fortsetzungsbereich), um sie besser lesbar und verwaltbar zu machen.

Werttyp

Muss entweder REG_SZ, REG_EXPAND_SZ REG_MULTI_SZ, REG_DWORD oder REG_BINARY sein.

Werttyp kann nur weggelassen werden, wenn KeyName weggelassen wird und das aktuelle Element der Registry-Schleife ein Wert ist, wie oben erwähnt.

KeyName

Der vollständige Name des Registry-Keys.

Dieser Parameter muss mit HKEY_LOCAL_MACHINE, HKEY_USERS, HKEY_CURRENT_USER, HKEY_CLASSES_ROOT oder HKEY_CURRENT_CONFIG beginnen (oder jeweils die Abkürzung, wie z. B. HKLM). Um den Zugriff auf eine Remote-Registry zu ermöglichen, muss der Computername und ein Doppelpunkt wie folgt vorangestellt werden: \\workstation01\HKEY_LOCAL_MACHINE

KeyName kann nur weggelassen werden, wenn eine Registry-Schleife läuft - in diesem Fall gilt standardmäßig der Key des aktuellen Schleifenelements. Wenn das Element ein Subkey ist, wird standardmäßig der vollständige Name dieses Subkeys benutzt. Wenn das Element ein Wert ist, gilt für Werttyp und Wertname standardmäßig der Typ und Name dieses Wertes, was aber überschrieben werden kann.

Wertname

Der Name des Wertes, der geschrieben werden soll. Fehlt dieser Parameter ist er leer, wird der Standardwert des Keys benutzt (außer wie oben erwähnt). Der Standardwert wird im RegEdit-Tool als "(Standard)" angezeigt.

ErrorLevel

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

A_LastError wird auf das Ergebnis der Systemfunktion GetLastError() gesetzt.

Bemerkungen

Wenn KeyName einen nicht-existierenden Subkey spezifiziert, wird RegWrite versuchen ihn zu erstellen (zusammen mit seinen übergeordneten Keys, falls nötig). RegWrite kann etwas direkt in einen Rootkey schreiben. Es gibt aber Betriebssysteme, die sich weigern, etwas in die oberste Ebene von HKEY_CURRENT_USER zu schreiben.

Wenn Werttyp auf REG_DWORD gesetzt wurde, sollte Wert zwischen -2147483648 und 4294967295 (0xFFFFFFFF) sein.

Beim Schreiben eines REG_BINARY-Keys muss eine Zeichenkette mit hexadezimalen Zeichen verwendet werden - z. B. kann ein REG_BINARY-Wert von 01,a9,ff,77 mit der Zeichenkette 01A9FF77 geschrieben werden.

Beim Schreiben eines REG_MULTI_SZ-Keys musst du jede Komponente mit einem Zeilenumbruchzeichen (`n) trennen. Bei der letzten Komponente kannst du das Zeilenumbruchzeichen weglassen. Leere Komponenten sind nicht erlaubt. Das heißt, dass du zwei Zeilenumbruchzeichen nicht nebeneinander angeben darfst (`n`n), weil das einen Wert in die Registry schreiben würde, der kürzer ist als erwartet.

Mit der Registry-Schleife können mehrere Registry-Keys und -Werte abgerufen und bearbeitet werden.

In den Bemerkungen zur Registry-Schleife findest du Hinweise, wie man auf die Registry eines Remotecomputers zugreifen kann.

Mit SetRegView können Einträge von den 64-Bit-Bereichen der Registry in einem 32-Bit-Skript oder umgekehrt gelesen und geschrieben werden.

Beispiele

RegWrite "Testwert", "REG_SZ", "HKEY_LOCAL_MACHINE\SOFTWARE\TestKey", "MeinWertname"
RegWrite "01A9FF77", "REG_BINARY", "HKEY_CURRENT_USER\Software\TEST_APP", "TEST_NAME"
RegWrite "Zeile1`nZeile2", "REG_MULTI_SZ", "HKEY_CURRENT_USER\Software\TEST_APP", "TEST_NAME"
