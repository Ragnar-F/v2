WinClose

Schließt das angegebene Fenster.

WinClose Fenstertitel, Fenstertext, Wartezeit, Titelausnahme, Textausnahme

Parameter

Fenstertitel

Ein Fenstertitel oder sonstige Kriterien, die das Zielfenster ermitteln sollen. Siehe Fenstertitel.

Fenstertext

In diesem Parameter kann eine Zeichenkette angegeben werden, die in einem Textelement des Zielfensters vorkommt (abrufbar mit Windows Spy). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.

Wartezeit

Fehlt dieser Parameter oder ist er leer, wartet die Funktion überhaupt nicht. Ist er 0, wartet der Befehl 500 ms. Ansonsten wartet der Befehl die angegebene Sekunden (auch mit Dezimalpunkt oder als Ausdruck), bis das Fenster sich schließt. Schließt sich das Fenster innerhalb dieser Zeit nicht, wird das Skript fortgesetzt. ErrorLevel wird nicht gesetzt, demzufolge kannst du WinExist oder WinWaitClose verwenden, um ein eindeutiges Schließen eines Fensters festzustellen. Während die Funktion in einem Wartezustand ist, können neue Threads mittels Hotkey, benutzerdefinierte Menüeinträge oder Timer gestartet werden.

Titelausnahme

Ignoriert Fenster, deren Titel diesen Wert enthalten.

Textausnahme

Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

Bemerkungen

Diese Funktion sendet eine Close-Meldung an das Fenster. Das Ergebnis hängt vom Fenster ab (es könnte fragen, ob du die Daten speichern willst, und so weiter).

Ist ein übereinstimmendes Fenster aktiv, wird vorrangig dieses Fenster geschlossen, anstatt ein anderes übereinstimmende Fenster darunter. Im Falle mehrerer übereinstimmender Fenster wird generell das oberste (aktuell verwendete) Fenster geschlossen.

Diese Funktion bezieht sich immer auf das oberste übereinstimmende Fenster; enthält Fenstertitel allerdings ahk_group Gruppenname, sind alle Fenster in der Gruppe betroffen.

WinClose sendet eine WM_CLOSE-Meldung an das Zielfenster, was eine etwas gewaltsame Methode zum Schließen des Fensters ist. Eine alternative Methode zum Schließen eines Fensters ist das Senden der folgenden Meldung. Beachte aber, dass das Fenster sich dabei anders verhalten kann, weil diese Methode quasi das gleiche ist wie, als würde man ALT+F4 oder den Schließen-Button in der Titelleiste drücken:

PostMessage 0x112, 0xF060,,, Fenstertitel, Fenstertext  ; 0x112 = WM_SYSCOMMAND, 0xF060 = SC_CLOSE

Wenn sich ein Fenster via WinClose nicht schließen lässt, kannst du es mit WinKill dazu zwingen.

Fenstertitel und -texte unterscheiden zwischen Groß- und Kleinschreibung. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.
