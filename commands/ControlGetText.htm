ControlGetText

Ermittelt den Text eines Steuerelements.

AusgabeVar := ControlGetText(Steuerelement, Fenstertitel, Fenstertext, Titelausnahme, Textausnahme)

Parameter

AusgabeVar

Name der Variable, in der der ermittelte Text gespeichert werden soll.

Steuerelement

Kann entweder das ClassNN (Klassenname und Instanznummer des Steuerelements) oder der Text des Steuerelements sein, das beides mit Windows Spy ermittelt werden kann. Verwendet man Text, ist das Übereinstimmungsverhalten abhängig von SetTitleMatchMode. Fehlt dieser Parameter oder ist er leer, wird das oberste Steuerelement des Fensters benutzt.

Um ein Steuerelement per HWND (Windows Handle) anzusteuern, muss der Steuerelement-Parameter leer gelassen und "ahk_id " SteuerelementHwnd im Fenstertitel-Parameter angegeben werden (funktioniert auch bei versteckten Steuerelementen, selbst wenn DetectHiddenWindows deaktiviert ist). Das HWND eines Steuerelements wird üblicherweise per ControlGetHwnd, MouseGetPos oder DllCall abgerufen.

Fenstertitel

Ein Fenstertitel oder sonstige Kriterien, die das Zielfenster ermitteln sollen. Siehe Fenstertitel.

Fenstertext

In diesem Parameter kann eine Zeichenkette angegeben werden, die in einem Textelement des Zielfensters vorkommt (abrufbar mit Windows Spy). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.

Titelausnahme

Ignoriert Fenster, deren Titel diesen Wert enthalten.

Textausnahme

Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

ErrorLevel

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Bemerkungen

Hinweis: Um Text aus einem ListView-, ListBox- oder ComboBox-Steuerelement zu ermitteln, verwende stattdessen ControlGetList.

Erscheint der ermittelte Text gekürzt (unvollständig), muss VarSetCapacity(AusgabeVar, 55) vor ControlGetText verwenden werden [55 sollte durch eine wesentlich größere Zahl als die Länge des Textes ersetzt werden]. Da einige Anwendungen nicht richtig auf die WM_GETTEXTLENGTH-Meldung reagieren, macht AutoHotkey die Ausgabevariable zu klein, wodurch der Text nicht mehr reinpasst.

Diese Funktion kann eine große Menge an RAM erzeugen, wenn das Steuerelement (z. B. ein Editor mit einem geöffneten umfangreichen Dokument) eine große Menge an Text beinhaltet. Allerdings kann der Speicher der Variable geleert werden, indem man ihr nichts zuteilt, z. B. AusgabeVar := "".

Meistens enthält der abgerufene Text eines Steuerelements CR- und LF-Zeichen (`r`n), anstatt ein einzelnes LF-Zeichen (`n), um das Ende jeder Zeile zu kennzeichnen.

Es ist nicht notwendig, SetTitleMatchMode "Slow" zu verwenden, weil ControlGetText den Text immer mithilfe der langsamen Methode ermittelt (da bei dieser Methode die Chance höher ist, dass der Befehl bei den meisten Steuerelementarten funktioniert).

Mit WinGetControls oder WinGetControlsHwnd kann eine Liste mit Steuerelementen von einem Fenster abgerufen werden.

Fenstertitel und -texte unterscheiden zwischen Groß- und Kleinschreibung. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.
