WinWait[Not]Active

Wartet, bis ein bestimmtes Fenster aktiv oder nicht aktiv ist.

WinWaitActive Fenstertitel, Fenstertext, Sekunden, Titelausnahme, Textausnahme
WinWaitNotActive Fenstertitel, Fenstertext, Sekunden, Titelausnahme, Textausnahme

Parameter

Fenstertitel

Typ: Zeichenkette

Der Titel oder andere Möglichkeiten zum Identifizieren des Zielfensters. Weitere Informationen finden Sie unter Fenstertitel-Parameter.

Fenstertext

Typ: Zeichenkette

Eine Zeichenkette, die in einem der Textelemente des Zielfensters enthalten ist (eventuell vorhandene Textelemente können mit dem internen Tool Window Spy aufgedeckt werden). Versteckte Textelemente werden nur erkannt, wenn DetectHiddenText eingeschaltet ist.

Sekunden

Typ: Integer oder Float

Die zu wartenden Sekunden, bevor eine Zeitüberschreitung erfolgt und 0 zurückgegeben wird. Lassen Sie diesen Parameter weg oder leer, um eine unbegrenzte Wartezeit zu ermöglichen. Die Angabe von 0 ist das gleiche wie, als würde man 0.5 angeben.

Titelausnahme

Typ: Zeichenkette

Alle Fenster, die diesen Wert in ihren Titeln enthalten, werden ignoriert.

Textausnahme

Typ: Zeichenkette

Alle Fenster, die diesen Wert in ihren Textelementen enthalten, werden ignoriert.

Rückgabewert

Typ: Integer

WinWaitActive gibt die HWND-Nummer (eindeutige ID) des aktiven Fensters zurück, wenn es mit den Kriterien übereinstimmt, oder eine 0, wenn die Funktion das Zeitlimit überschritten hat.

WinWaitNotActive gibt eine 1 zurück, wenn das aktive Fenster nicht mit den Kriterien übereinstimmt, oder eine 0, wenn die Funktion das Zeitlimit überschritten hat.

Bemerkungen

Die Funktion hört bereits auf zu warten, wenn ein aktives Fenster gefunden wird, und nicht erst, wenn die Sekunden abgelaufen sind. In diesem Fall wird sofort die Ausführung des Skripts fortgesetzt.

Da "A" mit dem jeweils aktiven Fenster übereinstimmt, wartet WinWaitNotActive "A" normalerweise unendlich lang. Um stattdessen zu warten, bis ein anderes Fenster aktiv wird, gibt man dessen eindeutige ID wie im folgenden Beispiel an:

WinWaitNotActive "ahk_id " WinExist("A")

Sowohl WinWaitActive als auch WinWaitNotActive werden das zuletzt gefundene Fenster aktualisieren, wenn zu Beginn der Funktion ein passendes Fenster aktiv ist oder wenn eines aktiv wird, während die Funktion wartet.

Während die Funktion im Wartezustand ist, können neue Threads via Hotkeys, benutzerdefinierte Menüpunkte oder Timer gestartet werden.

Wenn ein anderer Thread die Inhalte von Variablen, die die Funktion als Parameter verwendet, zwischendurch ändert, wird die Funktion diese Änderung ignorieren; das heißt, dass die Funktion weiterhin den Titel und Text verwendet, die ursprünglich in den Variablen vorhanden waren, als die Funktion zum ersten Mal zu warten begann.

Fenstertitel und -texte sind Groß-/Kleinschreibung-sensitiv. Versteckte Fenster werden nur erkannt, wenn DetectHiddenWindows eingeschaltet ist.

Beispiele

#1

Run "notepad.exe"
if WinWaitActive("Unbenannt - Editor", , 2)
    WinMinimize  ; Minimiert das von WinWaitActive gefundene Fenster.
else
    MsgBox "WinWaitActive hat das Zeitlimit überschritten."
