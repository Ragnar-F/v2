SetWinDelay

Setzt eine Verzögerung, die nach jeder fenster-spezifischen Funktion wie WinActivate erfolgen soll.

SetWinDelay Verzögerung

Parameter

Verzögerung

Zeit in Millisekunden. Verwenden Sie -1, um die Verzögerung vollständig zu deaktivieren, oder 0, um die kleinst mögliche Verzögerung zu veranlassen. Fehlt dieser Parameter, gilt standardmäßig 100.

Bemerkungen

Nach jeder fenster-spezifischen Funktion, außer WinActive und WinExist, erfolgt automatisch eine kurze Verzögerung (Ruhephase). Diese Verzögerung ist notwendig, um die Zuverlässigkeit der Skripte zu verbessern, weil einige Fenster eine Ruhephase benötigen, nachdem sie erstellt, aktiviert, minimiert usw. wurden. So haben sie eine Chance, sich selbst zu aktualisieren und auf die nächste Funktion reagieren zu können, die das Skript an dieses Fenster senden will.

Zwar ist eine Verzögerung von -1 erlaubt (überhaupt keine Verzögerung), allerdings wird empfohlen, mindestens 0 zu verwenden, um ein reibungsloses Ausführen des Skripts auch unter starker CPU-Belastung zu ermöglichen.

Eine Verzögerung von 0 führt intern ein Sleep(0) aus - diese Funktion übergibt den Restwert des Skript-Zeitintervalls an andere Prozesse, die diesen Wert benötigen. Wenn es solche Prozesse nicht gibt, wird Sleep(0) überhaupt keine Verzögerung durchführen.

Wenn die CPU langsam oder ausgelastet ist, oder die Fensteranimation aktiviert ist, könnten höhere Verzögerungswerte notwendig sein.

Die interne Variable A_WinDelay enthält die aktuelle Einstellung.

Jeder neu gestartete Thread (z. B. ein Hotkey, ein benutzerdefinierter Menüpunkt oder eine zeitgesteuerte Subroutine) beginnt mit der Standardeinstellung der Funktion. Um diese Standardeinstellung zu ändern, definiert man diese Funktion im automatischen Ausführungsbereich (der obere Bereich des Skripts).
