SetWorkingDir

Ändert das aktuelle Arbeitsverzeichnis des Skripts.

SetWorkingDir VerzName

Parameter

VerzName

Typ: Zeichenkette

Name des neuen Arbeitsverzeichnisses, das ein Unterordner im aktuellen A_WorkingDir-Verzeichnis ist, solange kein absoluter Pfad angegeben ist.

ErrorLevel

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Bemerkungen

Das Arbeitsverzeichnis des Skripts ist das Standardverzeichnis, das zum Zugreifen auf Dateien und Ordner verwendet wird, falls kein absoluter Pfad angegeben ist. Im folgenden Beispiel wird davon ausgegangen, das sich die Datei Dateiname.txt im A_WorkingDir-Verzeichnis befindet: FileAppend "Eine Textzeile.", "Dateiname.txt".

Das Arbeitsverzeichnis des Skripts ist standardmäßig immer A_ScriptDir, egal wie das Skript gestartet wurde. A_InitialWorkingDir ist hingegen das Arbeitsverzeichnis, das durch den Start des Skripts bestimmt wird. Startet man das Skript beispielsweise über eine Verknüpfung (z. B. via Startmenü), wird sein anfängliches Arbeitsverzeichnis durch das Feld "Ausführen in" im Eigenschaftsfenster dieser Verknüpfung bestimmt.

Diese Änderung wirkt sich sofort auf das gesamte Skript aus. Alle unterbrochenen, pausierten und neu ausgeführten Threads sind davon betroffen, einschließlich Timer.

Siehe auch

A_WorkingDir, A_InitialWorkingDir, A_ScriptDir, DirSelect

Beispiel

SetWorkingDir A_ScriptDir
SetWorkingDir "D:\Mein Ordner\Temp"
