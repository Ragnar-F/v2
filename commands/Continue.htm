Continue

Überspringt den Rest der aktuellen Schleife und beginnt einen neuen Durchlauf. Nur in einer Schleife gültig.

Continue LoopLabel

Wenn angegeben, kennzeichnet LoopLabel, für welche Schleife dieser Befehl gilt. Es kann entweder der Labelname oder die numerische Verschachtelungsebene angegeben werden. Ist dieser Parameter nicht vorhanden oder 1, gilt dieser Befehl für die innerste Schleife, von der er umschlossen ist. LoopLabel muss ein konstanter Wert sein, das heißt, dass Variablen und Ausdrücke nicht unterstützt werden. Wenn ein Label angegeben wurde, muss dieser direkt auf eine Loop-Anweisung verweisen.

Der Continue-Befehl verhält sich so, als würde er die geschweifte Endklammer einer Schleife erreichen:

  1. Er erhöht A_Index um 1.
  2. Er überspringt den Rest des Schleifenkörpers.
  3. Die Bedingung einer Schleife (falls vorhanden) wird dahingehend untersucht, ob sie erfüllt wurde. Wenn das der Fall ist, beginnt ein neuer Durchlauf; ansonsten endet die Schleife.

Break und Continue sind besser geeignet als Goto, weil sie ein Skript üblicherweise besser lesbar und verwaltbar machen.

Beispiel

; Dieses Beispiel zeigt vom 6. bis zum 10. Durchlauf 5 Dialogfenster.
; Bei den ersten 5 Durchläufen veranlasst die Continue-Anweisung
; immer einen Neustart des Durchlaufs, bevor die MsgBox-Zeile erreicht wird.
Loop 10
{
    if A_Index <= 5
        continue
    MsgBox A_Index
}
; Setzt die äußere Schleife fort.
outer:
Loop 3
{
    x := A_Index
    Loop 3
    {
        if (x*A_Index = 4)
            continue outer  ; Das gleiche wie continue 2 oder goto continue_outer.
        MsgBox x "," A_Index
    }
    continue_outer: ; Für goto.
