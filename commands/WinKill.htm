WinKill

Erzwingt das Schließen eines bestimmten Fensters.

WinKill Fenstertitel, Fenstertext, Wartezeit, Titelausnahme, Textausnahme

Parameter

Fenstertitel

Typ: Zeichenkette, Integer oder Objekt

Der Titel oder andere Möglichkeiten zum Identifizieren des Zielfensters. Weitere Informationen finden Sie unter Fenstertitel-Parameter.

Fenstertext

Typ: Zeichenkette

Eine Zeichenkette, die in einem der Textelemente des Zielfensters enthalten ist (eventuell vorhandene Textelemente können mit dem internen Tool Window Spy aufgedeckt werden). Versteckte Textelemente werden nur erkannt, wenn DetectHiddenText eingeschaltet ist.

Wartezeit

Typ: Integer oder Float

Wenn dieser Parameter weggelassen wird oder leer ist, wird die Funktion überhaupt nicht warten. Wenn eine 0 angegeben ist, wird standardmäßig eine Wartezeit von 500 ms verwendet. Jeder andere Wert wird als Sekunden interpretiert (auch mit Dezimalpunkt oder als Ausdruck) und definiert, wie lange auf das Schließen des Fensters gewartet werden soll. Schließt sich das Fenster innerhalb dieser Zeit nicht, wird das Skript fortgesetzt. ErrorLevel wird nicht gesetzt, demzufolge können Sie WinExist oder WinWaitClose verwenden, um ein eindeutiges Schließen eines Fensters festzustellen.

Titelausnahme

Typ: Zeichenkette

Alle Fenster, die diesen Wert in ihren Titeln enthalten, werden ignoriert.

Textausnahme

Typ: Zeichenkette

Alle Fenster, die diesen Wert in ihren Textelementen enthalten, werden ignoriert.

Bemerkungen

Zuerst versucht diese Funktion, das Fenster auf normale Weise zu schließen. Bei Misserfolg wird WinKill versuchen, den Prozess des Fensters gewaltsam zu beenden.

Wenn ein übereinstimmendes Fenster aktiv ist, wird bei mehreren übereinstimmenden Fenstern vorrangig dieses Fenster geschlossen. Im Falle mehrerer übereinstimmender Fenster wird generell das oberste (aktuell verwendete) Fenster geschlossen.

Diese Funktion agiert nur mit dem obersten übereinstimmenden Fenster, es sei denn, Fenstertitel ist ahk_group Gruppenname, dann sind alle Fenster in der Gruppe betroffen.

Fenstertitel und -texte sind Groß-/Kleinschreibung-sensitiv. Versteckte Fenster werden nur erkannt, wenn DetectHiddenWindows eingeschaltet ist.
