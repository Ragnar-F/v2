Sprachsyntax

Wann werden im Zusammenhang mit Befehlen und Parametern Anführungszeichen verwendet?

Anführungszeichen (") haben nur innerhalb von Ausdrücken eine besondere Bedeutung. Überall woanders werden sie literal behandelt, als wären sie ganz normale Zeichen. Führt ein Skript ein Programm oder ein Dokument aus, benötigt das Betriebssystem allerdings Anführungszeichen bei Befehlszeilenparameter, die Leerzeichen enthalten. Zum Beispiel: Run, Notepad.exe "C:\Meine Dateien\Adressenliste.txt".

Wann genau werden Variablen mit Prozentzeichen umschlossen (%)?

1) Variablennamen und andere Ausdrücke innerhalb von Zeichenketten (Text) werden immer mit Prozentzeichen umschlossen. Zum Beispiel:

MsgBox, Hallo`, %A_UserName%!
MsgBox("Schönen Tag, %A_UserName%!")

2) Variablennamen können auch mit Prozentzeichen umschlossen werden, um eine Doppeldereferenzierung zu erzielen. In solchen Fällen muss die Variable den Namen einer anderen Variable enthalten.

Abgesehen von Doppeldereferenzierungen und Zeichenketten gibt es Fälle, wo Prozentzeichen nicht benutzt werden (fett hervorgehoben):

1) Bei Parametern, die Ausgabevariablen sind: StrLen, AusgabeVar, %EingabeVar%
2) Überall in Ausdrücken. Zum Beispiel:

If (Var1 <> Var2)
    Var1 := Var2 + 100
MsgBox("Der Wert ist " Var1)

Wann sollten Prozentzeichen und Kommas mit einem Escapezeichen versehen werden?

Literale Prozentzeichen müssen mit einem Escapezeichen versehen werden, durch Voranstellen eines Akzents / umgekehrten Häkchens. Zum Beispiel:

MsgBox Der aktuelle Prozentsatz ist 25`%.
MsgBox("Der aktuelle Prozentsatz ist 25`%.")

Literale Kommas müssen auch mit einem Escapezeichen versehen werden (`,), außer bei einer MsgBox (in diesem Fall ist das umgekehrte Häkchen optional).

Sind Kommas innerhalb eines Ausdrucks in Anführungszeichen gesetzt, ist das umgekehrte Häkchen optional. Zum Beispiel: MsgBox("Hallo, Welt!").

Allgemeine Problembehandlung

Was kann ich tun, wenn AutoHotkey nicht installiert wird?

7-Zip-Fehler: Verwende 7-Zip oder ein kompatibles Programm, um die Setup-Dateien aus der Installer-EXE zu extrahieren und starte dann setup.exe oder Installer.ahk (zieh Installer.ahk mit gedrückter Maustaste auf AutoHotkeyU32.exe und lass los).

Der AutoHotkey-Installer ist ein selbstextrahierendes 7-Zip-Archiv, das versuchen wird, die Dateien in den temporären Benutzerverzeichnis zu entpacken und ein kompiliertes Skript zu starten. Manchmal kann es vorkommen, dass die Dateien aufgrund von Systemrichtlinien oder andere Faktoren nicht extrahiert oder ausgeführt werden dürfen. In solchen Fällen wird üblicherweise ein "7-Zip-Fehler" angezeigt. Die Dateien manuell in ein anderes Verzeichnis zu extrahieren könnte helfen.

Setup hängt: Erscheint das Setup-Fenster ohne Inhalt oder überhaupt nicht, kannst du eine oder beide folgende Methoden ausprobieren:

  • Halte STRG oder UMSCHALT gedrückt, wenn du den Installer startest. Wenn du eine Benutzerkontensteuerungsnachricht bekommst, musst du STRG oder UMSCHALT gedrückt halten, während du auf Ja/Fortfahren klickst. Jetzt sollte die Abfrage kommen, ob die Installation mit den Standardoptionen fortgesetzt werden soll.
  • Installiere AutoHotkey mithilfe der Kommandozeilenoptionen. Sobald du die Setup-Dateien manuell aus der Installer-EXE extrahiert hast, kannst du entweder setup.exe /S oder AutoHotkeyU32.exe Installer.ahk /S verwenden.

Sonstiges: Die obigen Vorschläge decken die häufigsten Probleme. Weitere Unterstützung findest du im Forum.

Warum werden einige Zeilen in meinem Skript nie ausgeführt?

Jede Zeile, die unmittelbar beim Starten des Skripts ausgeführt werden soll, muss ganz oben im Skript vorkommen und vor dem ersten Hotkey, Hotstring oder Return stehen. Siehe automatischer Ausführungsbereich für weitere Details.

Bei einem Hotkey, der mehr als eine Zeile ausführt, muss seine erste Zeile unterhalb des Hotkeys geschrieben werden, nicht auf der gleichen Zeile. Zum Beispiel:

#space::  ; WIN+LEERTASTE
Run Notepad
WinWaitActive Unbenannt - Editor
WinMaximize
return

Warum funktioniert mein Skript nicht in Windows xxx, obwohl es in einer vorherigen Version lief?

Es gibt mehrere Varianten von diesem Problem:

  • Ich habe mein Computer/Windows aufgewertet und stelle fest, dass mein Skript nicht mehr funktioniert.
  • Hotkeys/Hotstrings funktionieren nicht, wenn ein Programm mit Administratorrechten aktiv ist.
  • Einige Fenster weigern sich automatisiert zu werden (z. B. Geräte-Manager ignoriert Send).

Wenn du dein Betriebssystem wechselst, können sich auch Dinge ändern, die Einfluss auf dein Skript haben. Wenn du zum Beispiel einen neuen Computer hast, können andere Treiber oder Software installiert sein. Wenn du AutoHotkey auf eine neue Version aktualisiert hast, finde heraus, welche Version zuvor installiert war, und überprüfe dann die Changelog und Kompatibilitätshinweise auf Änderungen, die dein Skript stören könnten.

SoundGet, SoundSet, SoundGetWaveVolume und SoundSetWaveVolume verhalten sich in Windows Vista und höher anders als in früheren Versionen von Windows. Insbesondere sind die Gerätenummern unterschiedlich und einige Komponenten möglicherweise nicht verfügbar. Das Verhalten hängt von den Audio-Treibern ab, die sich zwangsläufig von den Treibern für Windows XP unterscheiden. Das Soundkartenanalyse-Skript kann verwendet werden, um die korrekten Gerätenummern ermitteln zu können.

Wie umgehe ich Probleme, die von der Benutzerkontensteuerung (UAC) erzeugt werden?

Benutzerkontensteuerung (UAC) ist eine häufige Ursache von Problemen, besonders beim Wechsel von Windows XP/Vista/7 zu Vista/7/8/10. UAC gibt es seit Windows Vista und ist standardmäßig auf neuen Systemen oder Installationen aktiviert. In Windows 8 und höher lässt sich UAC schwieriger deaktivieren.

Im Normalfall wird UAC ein Programm mit "erhöhter Berechtigung" (also ein Programm, das mit Administratorrechten läuft) davor schützen, von einem Programm ohne erhöhter Berechtigung automatisiert zu werden, weil es dadurch Sicherheitseinschränkungen umgehen könnte. Hotkeys werden auch blockiert. So zum Beispiel wäre ein Programm ohne erhöhter Berechtigung nicht in der Lage, Tastatureingaben auszuspionieren, die für ein Programm mit erhöhter Berechtigung gedacht sind.

UAC könnte auch verhindern, dass SendPlay und BlockInput funktionieren.

Als Übergangslösungen gibt es folgende:

  • Modifiziere AutoHotkey.exe, damit Skripte mit administrativen Programmen interagieren können. Allerdings hat das Nachteile. Für mehr Information, siehe http://www.autohotkey.com/board/topic/70449-enable-interaction-with-administrative-programs/
  • Führe das Skript als Administrator aus. Beachte, dass dadurch Programme, die vom Skript aus gestartet werden, auch als Administrator ausgeführt werden, und dass der Benutzer womöglich eine Zulassungsabfrage beim Starten des Skripts bestätigen muss.
  • Deaktiviere die lokale Sicherheitseinstellung "Alle Administratoren im Administratorgenehmigungsmodus ausführen" (nicht empfohlen).
  • Deaktiviere UAC vollständig. Dieser Schritt ist nicht empfohlen, und seit Windows 8 nicht mehr machbar.

Ich kann mein Skript nicht über das Infobereichssymbol editieren, weil es sich aufgrund eines Fehlers nicht starten lässt. Was soll ich tun?

Du musst den Fehler in deinem Skript beheben, wenn du das Infobereichssymbol zurück haben willst. Aber zuerst musst du die Skript-Datei finden.

Suche AutoHotkey.ahk in den folgenden Verzeichnissen:

  • Deine Dokumente (oder Eigene Dokumente).
  • Das Verzeichnis, in dem AutoHotkey installiert wurde, üblicherweise C:\Programme\AutoHotkey. Verwendest du eine nicht-installierte Version von AutoHotkey, dann das Verzeichnis, das AutoHotkey.exe enthält.

Wenn du eine AutoHotkey-Datei direkt ausführst, hängt der Name des Skripts von dieser Datei ab. Führt man beispielsweise AutoHotkeyU32.exe aus, wäre der Name des zu suchenden Skripts AutoHotkeyU32.ahk. Beachte, dass je nach Systemeinstellung die .ahk-Endung versteckt sein könnte, allerdings sollte die Datei ein Symbol haben, das wie folgt aussieht:

Ein Skript editiert man üblicherweise, indem man es mit der rechten Maustaste anklickt und Edit Script auswählt. Wenn das nicht funktioniert, kannst du die Datei in einem beliebigen Editor öffnen.

Wenn du AutoHotkey über das Startmenü startest oder wenn du AutoHotkey.exe direkt ausführst (ohne Befehlszeilenparameter), wird es nach einem Skript suchen, das in den oben genannten Standorten vorkommt. Alternativ kannst du ein Skript erstellen (irgendetwas.ahk), wo auch immer du es haben möchtest, und dann stattdessen dieses Skript ausführen.

Siehe auch Befehlszeilenparameter "Skript-Dateiname" und Portabilität von AutoHotkey.exe.

Wie kann ich Fehler in meinem Code finden und beheben?

Warum ist der Run-Befehl nicht in der Lage, mein Spiel oder Programm auszuführen?

Einige Programme müssen in ihren eigenen Verzeichnissen gestartet werden (im Zweifelsfall der sicherste Weg). Zum Beispiel:

Run, %ProgramFiles%\Beliebige Anwendung\Anwendung.exe, %ProgramFiles%\Beliebige Anwendung

Wenn du versuchst, ein Programm zu starten, das im Ordner %A_WinDir%\System32 liegt, und AutoHotkey 32-bit auf einem 64-Bit-System verwendest, dann könntest du Probleme mit dem File System Redirector bekommen. Du kannst ihn umgehen, wenn du stattdessen %A_WinDir%\SysNative verwendest; dieses virtuelle Verzeichnis können nur 32-Bit-Programme sehen, die auf einem 64-Bit-System laufen.

Warum funktionieren Hotstrings, Sends und Clicks in einigen Spielen nicht?

AHK kann nicht bei allen Spielen Tasten/Mausklicks senden oder Pixelfarben lesen.

Allerdings gibt es einige Alternativen - probiere alle unten erwähnten Lösungen aus. Wenn sie alle fehlschlagen, könnte es sein, dass AHK nicht für dein Spiel geeignet ist. Manche Spiele haben Maßnahmen integriert, um Hacks und Cheats zu verhindern, wie z. B. GameGuard und Hackshield. Wenn sie so etwas haben, ist es sehr wahrscheinlich, dass AutoHotkey nicht für dein Spiel funktionieren wird.

  • Verwende SendPlay via SendPlay-Befehl, SendMode Play und/oder Hotstring-Option SP.

    SendPlay, abc
    SendMode, Play
Send, abc
    :SP:bzw::beziehungsweise

; oder

#Hotstring SP
::bzw::beziehungsweise

    Hinweis: Es kann sein, dass SendPlay überhaupt nicht in Windows Vista oder höher funktioniert, wenn die Benutzerkontensteuerung (UAC) aktiv ist, selbst wenn das Skript als Administrator läuft.

  • Erhöhe SetKeyDelay. Zum Beispiel:

    SetKeyDelay, 0, 50
SetKeyDelay, 0, 50, Play

  • Probiere ControlSend, das dort funktionieren könnte, wo andere Sendemethoden versagen:

    ControlSend,, abc, Spiel_Titel

  • Probiere das Down- und Up-Ereignis einer Taste mit verschiedenen Sendemethoden:

    Send {TASTE down}{TASTE up}

  • Probiere das Down- und Up-Ereignis einer Taste mit einem Sleep dazwischen:

    Send {TASTE down}
Sleep 10 ; probiere verschiedene Millisekunden
Send {TASTE Up}

Wie kann die Performance für Spiele oder zu einem anderen Zeitpunkt verbessert werden, wenn die CPU stark belastet wird?

Sind Hotkeys, Clicks oder Sends im Skript bei starker CPU-Belastung spürbar langsamer als normal, könnte es hilfreich sein, die Priorität des Skript-Prozesses zu erhöhen. Füge dafür die folgende Zeile ganz oben im Skript ein:

ProcessSetPriority, High

Mein Antivirus-Programm hat AutoHotkey oder ein kompiliertes Skript als Malware erkannt. Ist das wirklich ein Virus?

Zwar kann nie ausgeschlossen werden, ob die Datei infiziert wurde, aber in den meisten Fällen sind solche Warnungen False-Positives (Falschmeldungen); das heißt, dass das Antivirus-Programm falsch liegt. Häufig wird vorgeschlagen, die Datei bei einem Online-Service wie virustotal oder Jotti hochzuladen und zu überprüfen, was andere Antivirus-Programme zu sagen haben. Im Zweifelsfall kannst du die Datei zum Hersteller deines Antivirus-Programms schicken und bestätigen lassen. Das würde auch uns und anderen AutoHotkey-Benutzern helfen, wenn der Hersteller die Datei als False-Positive anerkennt und ihr Produkt entsprechend anpassen.

False-Positives können häufiger auftreten, wenn kompilierte Skripte mit UPX (Standard in AutoHotkey 1.0, aber nicht in 1.1) oder MPRESS (optional in AutoHotkey 1.1) komprimiert sind. Da die Standardinstallation von AutoHotkey einen Kompressor nicht beinhaltet, werden kompilierte Skripte standardmäßig nicht komprimiert.

Wie kann die Ausgabe einer Befehlszeilenoperation abgefragt werden?

Tests zeigen, dass eine temporäre Datei aufgrund von Datei-Caching sehr schnell bei relativ kleinen Ausgaben sein kann. Tatsächlich wird sie gar nicht erst auf das Laufwerk geschrieben, wenn die Datei unmittelbar nach ihrer Verwendung gelöscht wird. Zum Beispiel:

RunWait %comspec% /c dir > C:\Meine Temp-Datei.txt
FileRead, AusgabeVar, C:\Meine Temp-Datei.txt
FileDelete, C:\Meine Temp-Datei.txt

Wenn du temporäre Dateien vermeiden willst (besonders wenn die Ausgabe groß ist), dann verwende die Shell.Exec()-Methode, welche in den Beispielen des Run-Befehls zu finden ist.

Wie kann ein Skript andere Skripte schließen, pausieren oder sperren?

Zunächst ein Beispiel, das ein anderes Skript schließt:

DetectHiddenWindows On  ; Ermöglicht die Erkennung des versteckten Hauptfensters eines Skripts.
SetTitleMatchMode 2  ; Verhindert, dass unten der vollständige Pfad der Datei angegeben werden muss.
WinClose Skript-Dateiname.ahk - AutoHotkey  ; Verwende hier den Namen deines Skripts (zeichengenau).

Um ein anderes Skript zu sperren oder zu pausieren, ersetze die oben genannte letzte Zeile mit eine der folgenden Zeilen:

PostMessage, 0x111, 65305,,, Skript-Dateiname.ahk - AutoHotkey  ; Suspend (Sperren).
PostMessage, 0x111, 65306,,, Skript-Dateiname.ahk - AutoHotkey  ; Pause.

Wie kann eine sich wiederholende Aktion gestoppt werden, ohne das Skript beenden zu müssen?

Um bei Tastendruck das ganze Skript zu pausieren oder fortzusetzen, weise dem Pause-Befehl einen Hotkey wie folgt zu:

^!p::Pause  ; Drücke STRG+ALT+P für Pause. Drückt es nochmal, um fortzufahren.

Um eine sich wiederholende Aktion innerhalb einer Schleife zu stoppen, verwende den folgenden Hotkey, der seine eigene wiederholende Aktion sowohl starten als auch beenden kann. Das heißt, dass durch einmaliges Drücken des Hotkeys die Schleife begonnen und durch nochmaliges Drücken diese wieder beendet wird.

#MaxThreadsPerHotkey 3
#z::  ; WIN+Z (passe diesen Hotkey nach Belieben an).
#MaxThreadsPerHotkey 1
if WinZLaufenLassen  ; Bedeutet, dass ein darunterliegender Thread bereits die untere Schleife ausführt.
{
    WinZLaufenLassen := false  ; Signalisiert dem Thread, die Schleife zu stoppen.
    return  ; Beendet diesen Thread, so dass der darunterliegende fortgesetzt wird und die vorgenommene Änderung der oberen Zeile sehen kann.
}
; Ansonsten:
WinZLaufenLassen := true
Loop
{
    ; Die nächsten vier Zeilen sind Aktionen, die mehrfach ausgeführt werden sollen (passe sie nach Belieben an):
    ToolTip, Drücke erneut WIN+Z, um das hier zu stoppen.
    Sleep 1000
    ToolTip
    Sleep 1000
    ; Lasse den Rest aber unverändert.
    if not WinZLaufenLassen ; Der Benutzer hat die Schleife durch erneutes Drücken von WIN+Z gestoppt.
        break  ; Schleife unterbrechen.
}
WinZLaufenLassen := false  ; Zurücksetzen, damit der Hotkey wieder verwendet werden kann.
return

Wie können Datums- oder Zeitwerte verglichen oder geändert werden?

Ergebnis := DateAdd(VariableMitZeitstempel, -7, "days")

Um die Zeitmenge zwischen zwei Datum- oder Zeitwerten zu ermitteln, siehe DateDiff. Die Built-in-Variable A_Now enthält die aktuelle Ortszeit. Außerdem gibt es noch andere built-interne Datum/Zeit-Variablen, sowie auch den FormatTime-Befehl, um eine benutzerdefinierte Datum/Zeit-Zeichenkette erstellen zu können.

Hotkeys, Hotstrings und Neubelegung

Wie können meine Hotkeys und Hotstrings automatisch bei jedem PC-Start ausgeführt werden?

Es gibt einen Ordner im Startmenü, der sich "Autostart" nennt. Fügst du eine Verknüpfung zu deinem Skript in diesen Ordner ein, wird es jedes Mal gestartet, wenn du dein PC hochfährst. Um eine Verknüpfung zu erstellen:

  1. Finde die Skript-Datei, markiere sie und drücke STRG+C.
  2. Klicke mit der rechten Maustaste auf Start (üblicherweise in der linken unteren Ecke des Bildschirms) oder im Startmenü auf "Alle Programme" (falls das neue Design aktiv ist) und wähle "Öffnen - Alle Benutzer" aus.
  3. Navigiere zum Autostart-Ordner innerhalb des Programmordners.
  4. Wähle über die Menüleiste Bearbeiten -> Verknüpfung einfügen aus. Die Verknüpfung des Skripts sollte nun im Autostart-Ordner sein.

Ich habe Schwierigkeiten, dass meine Maustasten wie Hotkeys funktionieren. Irgendwelche Hinweise?

Die linke und rechte Maustaste sollte ganz normal zuweisbar sein (zum Beispiel wäre #LButton:: der Hotkey WIN+MAUSTASTE-LINKS). Das gilt auch für die mittlere Maustaste und das Mausrad, außer bei Mäusen, deren Tasten direkt vom Treiber gesteuert werden.

Die vierte (XButton1) und fünfte Maustaste (XButton2) sind möglicherweise auch zuweisbar, sofern dein Maustreiber diese Klicks für das System erkennbar macht. Sind sie nicht erkennbar (oder hat die Maus mehr als 5 Tasten, die verwendet werden sollen), kann man versuchen, die mitgelieferte Software der Maus manuell zu konfigurieren (meistens in der Systemsteuerung oder im Startmenü aufrufbar), um eine Tastatureingabe zu senden, wann immer man eine dieser Tasten drückt. So eine Tastatureingabe könnte dann in einem Skript als Hotkey definiert werden. Bringt man beispielsweise die vierte Maustaste dazu, STRG+F1 zu senden, könnte man sie als Hotkey verwenden, wenn ^F1:: im Skript definiert wird.

Wenn man eine 5-Tasten-Maus besitzt, deren vierte und fünfte Maustaste nicht erkennbar sind, kann man versuchen, den Maustreiber mit dem Standardtreiber des Betriebssystems zu ersetzen. Das setzt voraus, dass so ein Treiber für deine Maus vorhanden ist und dass du ohne den mitgelieferten Features von der Software leben kannst.

Wie können TAB und LEERTASTE als Hotkeys definiert werden?

Verwende nicht die Zeichen der Tasten, sondern ihre englischen Namen (Tab und Space). Zum Beispiel ist #Space WIN+LEERTASTE und ^!Tab STRG+ALT+TAB.

Wie können Tastatur- oder Maustasten neu belegt werden, so dass sie zu anderen Tasten werden?

Das wird auf der Seite "Neubelegung von Tasten" beschrieben.

Wie kann ich einen doppelten Tastendruck oder Mausklick erkennen?

Verwende die Built-in-Variablen für Hotkeys wie folgt:

~Ctrl::
    if (A_ThisHotkey = A_PriorHotkey && A_TimeSincePriorHotkey < 200)
        MsgBox Doppeldruck
return

Wie kann ein Hotkey oder Hotstring exklusiv für ein bestimmtes Programm bereitgestellt werden? Mit anderen Worten möchte ich eine bestimmte Taste dazu bringen, dass sie sich anders als normal verhält, falls ein bestimmtes Fenster aktiv ist.

Die bevorzugte Methode ist #IfWinActive. Zum Beispiel:

#IfWinActive, ahk_class Notepad
^a::MsgBox Sie haben STRG+A gedrückt, während der Editor aktiv ist.

Wie kann man das machen, dass eine Präfix-Taste ihre ursprüngliche Funktion durchführt, anstatt nichts zu tun?

Das folgende Beispiel macht Numpad0 zu einer Präfix-Taste:

Numpad0 & Numpad1::MsgBox, Sie haben Numpad1 gedrückt, während Numpad0 gedrückt wurde.

Um Numpad0 nun dazu zu bringen, immer dann eine echte Numpad0-Tastatureingabe zu senden, wenn sie nicht zum Starten eines Hotkeys wie oben benötigt wird, füge den folgenden Hotkey hinzu:

 $Numpad0::Send, {Numpad0}

Das $-Präfix ist notwendig, um eine Warnmeldung im Falle einer Endlosschleife zu unterdrücken (da der Hotkey "sich selbst sendet"). Darüber hinaus wird die obere Aktion erst ausgelöst, wenn die Taste losgelassen wird.
